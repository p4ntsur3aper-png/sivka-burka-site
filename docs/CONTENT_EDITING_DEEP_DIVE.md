# Углубление админ-редактирования контента

## Цель

Сделать админ-режим не отдельной "технической админкой", а режимом управления сайтом: администратор входит, переходит на любую публичную страницу и меняет текст, фото, порядок блоков и карточки прямо в контексте страницы.

## Текущее состояние

Реализовано:

- демо-вход администратора;
- переключатель режима редактирования;
- inline-редактирование hero-блока главной страницы;
- редактирование заголовков страниц через `EditablePageTitle`;
- загрузка изображения через `ImageUploadButton`;
- редактирование услуг, галереи, лошадей, заявок и правил через `/admin`;
- сохранение в `localStorage`;
- событие `orlov-content-updated` для обновления интерфейса.

Главное ограничение: контент пока не описан как единая система управляемых блоков. Часть текста находится в `SiteContent`, часть - в mock-данных, часть - прямо в JSX. Поэтому требование "редактировать все" невозможно гарантировать без отдельной модели контента.

## Что нужно считать "всем редактируемым"

Минимальный набор для дипломного проекта:

- название сайта и подпись бренда;
- пункты меню и подписи в футере;
- hero-блок главной: надзаголовок, заголовок, текст, фон, позиция фото, кнопки;
- блок популярных услуг: заголовок, текст, состав карточек, порядок карточек;
- преимущества на главной: иконка, заголовок, текст, порядок;
- блок доверия/безопасности на главной;
- отзывы на главной и странице отзывов;
- все тексты заголовков страниц;
- услуги: название, цена, длительность, фото, описание, ограничения, правила, порядок, доступность;
- карточка услуги: все текстовые секции и изображение;
- форма записи: поясняющие тексты, но не технические названия полей без необходимости;
- правила и FAQ: разделы, вопросы, ответы, порядок;
- галерея: альбомы, заголовки альбомов, описания, фото, порядок, обложки;
- лошади: фото, имя, описание, статус, вес, допустимые услуги, порядок отображения;
- контакты: адрес, телефон, email, режим обработки заявок, мессенджеры, ссылки, текст карты;
- SEO-поля: title, description, изображение для соцсетей.

## Предлагаемая CMS-модель на frontend

### ContentBlock

```ts
type ContentBlockType =
  | 'hero'
  | 'sectionTitle'
  | 'text'
  | 'benefits'
  | 'serviceList'
  | 'reviews'
  | 'galleryAlbum'
  | 'faq'
  | 'contacts'
  | 'cta';

interface ContentBlock {
  id: string;
  pageKey: string;
  type: ContentBlockType;
  title?: string;
  subtitle?: string;
  text?: string;
  imageId?: string;
  settings: Record<string, unknown>;
  order: number;
  isVisible: boolean;
}
```

## UX админ-режима

Рекомендуемая логика:

1. Администратор входит.
2. Вверху появляется панель: "Режим редактирования", "Предпросмотр", "Сохранить", "Опубликовать", "Выйти".
3. На публичных страницах у каждого редактируемого блока появляется компактная кнопка редактирования.
4. При клике открывается боковая панель, а не большой блок внутри страницы.
5. Для изображения используется не только поле URL, а кнопки:
   - выбрать из медиатеки;
   - загрузить файл;
   - заменить;
   - удалить изображение;
   - настроить позицию/crop.
6. Для списков появляются действия:
   - добавить элемент;
   - удалить;
   - скрыть;
   - поднять/опустить;
   - дублировать.
7. Сложные поля проверяются до сохранения.

## Что убрать или заменить

| Текущее решение | Что сделать лучше |
|---|---|
| Data URL в поле изображения | MediaAsset + upload endpoint |
| Ручной ввод `URL или gradient` | Кнопка выбора файла и медиатека, URL оставить как дополнительный вариант |
| JSON-конфиг правил записи | Формы под каждый тип правила |
| Сохранение только в `localStorage` | Mock repository сейчас, backend repository позже |
| Inline-панели внутри карточек | Боковая панель редактирования с предпросмотром |
| Отсутствие порядка элементов | Поле `order` и кнопки перемещения |

## Минимальный результат следующей итерации

Чтобы проект выглядел убедительно как бизнес-приложение, достаточно сделать CMS v1:

- `contentRepository.ts` как единый слой чтения/записи контента;
- типы `ContentBlock`, `MediaAsset`, `MediaFolder`;
- mock-медиатеку;
- боковую панель редактирования;
- редактирование всех текстовых блоков главной страницы;
- редактирование контактов, отзывов, правил и FAQ;
- загрузку файлов через общий компонент `MediaPicker`;
- сортировку услуг, фото, отзывов и блоков галереи;
- подтверждение удаления;
- отдельный документ API-контракта для backend.
