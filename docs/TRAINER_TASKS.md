## TASK-0605. Создать trainerRepository и notificationRepository

Приоритет: P0  
Роль: Frontend Engineer  
Файлы: `src/services/trainerRepository.ts`, `src/services/notificationRepository.ts`, `src/services/adminContent.ts`, `src/services/backendApi.ts`, `src/pages/AdminPage.tsx`

Что сделать:

- перед созданием уведомлений стабилизировать единый источник данных тренеров;
- убрать расхождение между `orlov_trainers` и `orlov_admin_trainers`;
- сделать так, чтобы админка, trainerRepository и будущий кабинет тренера читали и сохраняли один и тот же список тренеров;
- проверить, что изменения тренера в админке видны через `getTrainers`;
- исправить назначение тренера так, чтобы оно не затирало несохраненные изменения заявки в UI;
- вынести проверку допустимости назначения тренера в сервисный/API-слой, а не оставлять только UI-фильтр;
- запретить назначение тренера, если он неактивен, недоступен или не проводит выбранную услугу;
- создать функции для получения уведомлений;
- создать функцию `createNotification`;
- создать `markNotificationAsRead`;
- создать `getUnreadNotificationsCount`;
- создавать уведомление при назначении тренера.

Критерии готовности:

- у тренеров один mock storage key и один источник истины;
- `getEditableTrainers` и `trainerRepository` не расходятся по данным;
- назначение тренера не откатывает локальные изменения заявки в календаре;
- `assignBookingTrainer` проверяет активность тренера и `allowedServiceIds`;
- уведомления не захардкожены в компонентах;
- уведомление создается сервисом;
- можно заменить хранилище на backend API.
