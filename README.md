# Sivka-Burka site

Веб-приложение конно-спортивного клуба: публичный сайт, форма записи, административная панель, кабинет управляющего и кабинет тренера.

## Быстрый запуск на Windows

1. Установите Node.js 22 LTS или новее: https://nodejs.org/
2. Распакуйте проект в любую папку.
3. Дважды нажмите `start-local.bat`.
4. При первом запуске скрипт сам выполнит `npm install`.
5. После запуска откроется сайт, обычно по адресу `http://127.0.0.1:5174/`.

Скрипт запускает два окна:

- `Sivka-Burka Backend` - сервер, API и SQLite-база.
- `Sivka-Burka Frontend` - Vite-приложение в браузере.

Пока приложение используется, оба окна нужно держать открытыми. Чтобы остановить локальный запуск, закройте эти два окна.

## Запуск из терминала

```powershell
npm run start:local
```

Эта команда делает то же самое, что и `start-local.bat`: проверяет Node.js, ставит зависимости при необходимости, создает `.env.local`, выбирает свободные порты и запускает backend + frontend.

## Ручной запуск

Если автоматический запуск не подходит, можно запустить процессы вручную в двух терминалах.

Терминал 1:

```powershell
$env:BACKEND_PORT="8090"
npm run backend:dev
```

Терминал 2:

```powershell
$env:VITE_API_BASE_URL="http://127.0.0.1:8090/api"
npm run dev -- --host 127.0.0.1 --port 5174
```

После этого открыть `http://127.0.0.1:5174/`.

## Важные файлы

- `start-local.bat` - простой запуск двойным кликом.
- `scripts/start-local.ps1` - основной PowerShell-запускатель.
- `backend/src/server.js` - backend API.
- `backend/src/seed-data.js` - исходные данные для первого создания или сброса SQLite.
- `backend/.data/sivka_burka.sqlite` - локальная база данных после первого запуска.
- `.env.example` - пример переменных окружения для frontend.
- `docs/` - техническая документация по backend, API и демонстрации.

## Публикация на GitHub

Инструкция по загрузке проекта на GitHub находится в `GITHUB_UPLOAD.md`.
Для создания чистого архива без временных файлов можно выполнить:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/create-github-archive.ps1
```

## Проверка перед передачей проекта

Перед тем как отправлять проект другому человеку, лучше удалить папки и файлы, которые восстановятся автоматически:

- `node_modules/`
- `dist/`
- `.env.local`
- `backend/.data/`, если нужно отправить проект без накопленных заявок и изменений.

После распаковки получатель сможет снова запустить `start-local.bat`.
